1 Version 2026.5 (2026-06-05)

1.1 AnnoLyze Bug Fixes & Stability

1.1.1 Silent Crash on Folder Load with Config

  • Fixed a silent crash that occurred when loading an image folder containing a config.json
  • Root cause: configuration apply and data loading mutated Tkinter widgets from a background worker thread, triggering a non-catchable Tcl crash that bypassed the error handler
  • Image gathering now stays on the worker thread, while all UI work (image display, config/annotation/result loading) is marshalled back to the main thread
  • Real configuration errors now surface as proper error popups and status-bar messages instead of crashing silently

1.1.2 Thread-Safe Error Popups

  • show_error_popup now detects calls from worker threads and schedules the popup on the GUI thread, so error reporting can never itself crash the application

1.1.3 Logger Fix

  • Fixed an AttributeError when switching slices (annotatePanel referenced an undefined self.logger); replaced with a proper module-level logger

1.2 AnnoLyze User Experience

1.2.1 Clean Slate on New Folder Load

  • Loading a new folder now resets all previous state: results table, dynamic columns, annotations, cached config, and keybindings
  • Previously the old config, results, and annotations persisted into the newly loaded folder

1.2.2 Live Keybinding Activation

  • Keybindings defined in the GUI via Add Column are now active immediately, without needing to save and reload a config file first
  • Removing a column now deactivates its keybinding live
  • Introduced a single shared KeybindingManager per session so GUI-defined and config-loaded keybindings share the same live bindings

3.5 Performance Improvements

3.5.1 Faster OCT File Detection (RexView)

  • Sped up OCT file detection by parsing each scan’s XML header only once per file instead of once per metadata field
  • Previously, metadata extraction re-opened the OCT archive and re-parsed the entire Header.xml five times per file (one per value: dataType, Serialnumber, dimX, dimY, dimZ)
  • Added getXMLDict() for single-pass header parsing; getXMLvalue() now delegates to it
  • FileDiscoveryService now uses a single-pass xml_dict_reader, dramatically reducing detection time for folders with many OCT files

3.5.2 Parallel Export (RexView)

  • Export now processes multiple OCT files in parallel across CPU cores instead of strictly one after another
  • Each file is exported in its own worker process, which is well suited to the CPU-bound raw-spectral reconstruction (FFT, noise-floor, filtering)
  • Worker count defaults to CPU cores - 1 (leaving a core for the UI), bounded by the number of queued files, a hard cap, and an optional memory budget; it can be overridden via ExportConfig.worker_count
  • Removed a per-slice gc.collect() and now build the XML metadata dictionary only once per file, both of which speed up every export regardless of parallelism
  • Export status is reported per file (queued → Done / Done (N failed) / Error); UI updates are marshalled back to the main thread for thread safety

5 Version 2026.1 (2026-03-12)

5.1 Bug Fixes

5.1.1 Refractive Index Correction for COMBINED_MEAN

  • Fixed missing refractive index correction in the COMBINED_MEAN detection method path
  • The COMBINED_MEAN recomputation block was using raw optical pixel depths instead of applying the cavitation-aware refractive index correction (air gap at n=1 + subsurface at n=1.5)
  • This caused LESION_DEPTH_MEAN in the Excel Summary to report uncorrected (overestimated) depths when COMBINED_MEAN was selected
  • Now applies the same correction logic as all other detection methods (KNEE_POINT, SIGMOID_FIT, SIGMOID_SHOULDER)

8.1 Carl Quant Module - Algorithm Improvements

8.1.1 Buffer Zone System for Transition Areas

  • Implemented 10-pixel buffer zone around lesion boundaries to avoid transition artifacts
  • Added buffer_pixels field (default 10) to RegionConfig data model
  • Created helper methods for buffered coordinate calculations:
    • get_buffered_lesion_start_x(): Moves lesion start 10px right
    • get_buffered_lesion_end_x(): Moves lesion end 10px left
    • get_buffered_sound_left_end_x(): Ends sound area 1 at 10px before lesion
    • get_buffered_sound_right_start_x(): Starts sound area 2 at 10px after lesion
  • Updated analysis functions to use buffered coordinates:
    • Interpolated surface fitting excludes transition zones
    • Sound region extraction avoids transition zone contamination
    • Lesion depth detection uses buffered boundaries (10px shorter on each side)
    • Cavitation detection uses buffered boundaries for consistency
  • Visual boundaries remain at user-defined positions for clear feedback
  • Backward compatible with existing configurations (defaults to 10px buffer)
